# 沉浸式交互路线图

这个应用的卖点不是普通白噪音，也不是普通番茄钟，而是模拟“我真的离开了家，进入一个公共空间，并开始认真做事”的心理启动过程。

当前 MVP 已经包含：进入咖啡店、点饮品、选座位、设置任务、专注计时、环境音层。下一步要把它升级成更像第三人称沉浸式咖啡店体验。

## 产品边界

- 不做聊天产品。
- 不做开放式 AI 角色扮演。
- 不让视频承担交互逻辑。
- 不让提醒变成监控或责备。
- 所有插入事件都服务于一个目标：把用户轻轻带回当前任务，或者允许用户有边界地休息。

## 场景结构

建议新增一个更明确的场景状态：

```js
const scenes = [
  "entrance",
  "doorOpening",
  "ordering",
  "seatSelection",
  "walkingToSeat",
  "setup",
  "focus",
  "break",
  "complete",
];
```

对应体验：

1. `entrance`: 咖啡店外景循环，按钮是“推门进入”。
2. `doorOpening`: 主角推门进入的 2 到 3 秒过渡视频，自动进入点单。
3. `ordering`: 咖啡师在吧台后，网页叠加点单对话框和饮品按钮。
4. `seatSelection`: 镜头扫过不同座位，用户选择座位。
5. `walkingToSeat`: 主角走向对应座位的短过渡。
6. `setup`: 坐下，输入任务，完成“放好手机”“打开电脑”等动作。
7. `focus`: 专注桌面循环视频，保留最小化计时器、任务、环境音。
8. `break`: 休息短事件，咖啡师或旁桌顾客短暂出现。
9. `complete`: 结束总结。

## 第三人称仪式感

主角不需要一直出现在画面里，但关键动作要出现：

- 推门进入。
- 站在吧台前点单。
- 走向座位。
- 坐下，放包，打开电脑。
- 暂停太久时回神。
- 休息时拿起杯子或看向窗外。

这些动作是心理启动程序，不是剧情动画。每段都应短、安静、可跳过。

## Intentional 插入事件

插入事件分两类：停滞提醒和休息允许。

### 停滞提醒

触发条件建议：

- 用户在专注中暂停超过 90 秒。
- 用户离开 `focus` 状态后又回来。
- 用户主动暂停多次。

语气：

- 轻，不责备。
- 有公共场合感。
- 最后把注意力带回任务名。

示例：

```text
旁桌顾客：
刚才看你一直很认真。

我：
既然之前已经坐下来认真开始了，现在先不要散掉。回到“{task}”的下一小步。
```

如果坐在吧台：

```text
咖啡师：
你的杯子还在这边。慢慢来。

我：
我已经来到这里了。继续完成“{task}”，先做一个很小的步骤。
```

如果坐在安静区，不出现外部人物：

```text
我：
我不是在家里躺着。我已经在这里坐下了。现在回到“{task}”。
```

### 休息允许

触发条件建议：

- 已专注 25 分钟以上。
- 倒计时完成。
- 用户主动点击休息。

语气：

- 承认已经做了事。
- 允许休息。
- 不把休息设计成失败。

示例：

```text
咖啡师：
要不要再加一点热水？

我：
我已经专注了 {minutes} 分钟。现在休息一下是合理的。
```

窗边座位：

```text
窗边顾客翻过一页书。

我：
我已经和“{task}”待了一段时间。可以看一会儿窗外，然后再决定下一步。
```

## 座位与角色

| 座位 | 适合事件角色 | 声音重点 |
| --- | --- | --- |
| 窗边座位 | 窗边顾客 / 内心独白 | 雨声、轻街声、人声较轻 |
| 角落小桌 | 旁桌顾客 / 内心独白 | 低咖啡店环境、人声低 |
| 吧台座位 | 咖啡师 | 后厨咖啡声、杯子声、轻爵士 |
| 安静区 | 内心独白 | 稳定白噪音、极少人声 |

## 技术实现草案

新增状态：

```js
const [intervention, setIntervention] = useState(null);
const [pausedAt, setPausedAt] = useState(null);
const [focusedSeconds, setFocusedSeconds] = useState(0);
const [lastBreakPromptAt, setLastBreakPromptAt] = useState(0);
```

事件结构：

```js
const INTERVENTIONS = {
  pauseLong: {
    window: {
      speaker: "窗边顾客",
      line: "刚才看你一直很认真。",
      thought: (task) => `既然已经开始了，现在先回到“${task}”的下一小步。`,
    },
    bar: {
      speaker: "咖啡师",
      line: "你的杯子还在这边。慢慢来。",
      thought: (task) => `我已经来到这里了。继续完成“${task}”，先做一个很小的步骤。`,
    },
  },
  breakReady: {
    bar: {
      speaker: "咖啡师",
      line: "要不要再加一点热水？",
      thought: (minutes) => `我已经专注了 ${minutes} 分钟。现在休息一下是合理的。`,
    },
  },
};
```

显示方式：

- 在 `focus` 页面叠加一个轻量对话框。
- 对话框出现后暂停环境提示消息，不停止计时。
- 用户点“继续”后关闭对话框。
- 休息事件可以切到 `break` 场景，停留 30 到 90 秒。
